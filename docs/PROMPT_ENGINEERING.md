# SynapseIQ — Enterprise AI Prompt Engineering & Schema Specification

This document details the prompt engineering design, McKinsey/BCG persona constraints, domain adaptation logic, and JSON schema contracts powering **SynapseIQ's Intelligence Engine**.

---

## 🎯 1. Core AI Persona & System Constraints

All prompts generated by SynapseIQ enforce strict executive advisory persona constraints:

```typescript
Role: Executive Business Advisor (McKinsey / BCG persona).
Guidelines:
1. Speak concisely, analytically, and authoritatively. Avoid generic or repetitive phrasing.
2. Ground all insights in local biAnalysis metrics (covariance, slopes, Z-scores).
3. Focus on trends (slopes, MoM vectors) and covariance between metrics instead of isolated numbers.
4. Every strategic recommendation MUST output a stringified JSON object conforming to the NodeRecommendation contract.
```

## 🛡️ 4. Validation & Sanitization Loop

Before any Gemini response hydrates the Zustand global store, it passes through `cleanAndValidateAnalysis()` / `cleanAndValidateCopilot()`:

1. **Repetition Check**: Rejects responses with duplicate sentences.
2. **Phrase Filtering**: Rejects responses containing prohibited generic filler phrases (*"Based on the data..."*, *"As an AI language model..."*).
3. **JSON Structure Audit**: Ensures required keys exist and nested JSON recommendations parse cleanly.
4. **Auto-Regeneration**: If validation fails, the service auto-retries up to 2 times before falling back to local heuristic calculations.
